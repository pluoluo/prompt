GPT Image 2 提示词编写指南

来源：veo4.im

核心流程

想把 GPT Image 2 用好，要把它当成一个设计执行器，而不是魔法按钮。最实用的默认流程如下：

步骤先决定什么默认建议
1输出类型海报、产品视觉、信息图、UI 板、肖像或广告素材
2工作模式新场景用生成；已有底图接近时优先编辑
3提示词结构主体 → 版式 → 风格 → 光线 → 文字 → 投放渠道
4成功标准可读性、构图、真实感、品牌匹配度
5修改方式每轮只修一类问题，不要整段重写

一、先定义任务，再写 Prompt

不要一开始就写 beautiful、epic、stunning 这种空泛词。先问自己：这张图到底要完成什么工作？

目标最合适的起手方式为什么
社媒广告或海报以文字层级为核心的版式任务说明标题层级和风格同样重要
信息图或解释图先定义标签结构的解释图任务说明结构先清楚，才能避免装饰化废图
产品主视觉图写实产品场景任务说明表面材质、角度和光线决定质量
UI 板或设计概念模块化布局任务说明面板、间距、层次需要显式约束
人像或生活方式主体优先的电影感任务说明身份和光线应该比风格词更靠前

二、结构化 Prompt 模板

最稳定的提示词，靠的是结构，不是灵感。按以下顺序写：

模块需要写什么示例
输出类型海报、信息图、产品渲染、肖像、UI 板Create a launch poster
主体核心对象、人物或场景for a minimalist AI design studio
构图取景、层级、留白、角度centered layout with large headline space
风格编辑型、写实、插画风等clean editorial design
光线与材质光向、质感、表面、真实感soft side light, matte glass, subtle reflections
图中文字必须出现的精确文案"Design Faster with Better Control"
使用场景落地页、广告、博客头图、演示文稿for a landing-page hero

可直接套用的模板：

Create a [output type] featuring [main subject]. Use [composition and framing]. The style should feel [style direction]. Lighting and materials: [light, texture, surfaces]. Visible text: "[exact text]". Make it suitable for [delivery context]. Avoid clutter, unreadable text, extra objects, and off-brand colors.

三、生成 vs 编辑

很多团队浪费时间，是因为本该编辑的时候还在不断重生成。

模式适合什么情况常见用途
生成需要新场景、新布局、新方向首轮概念图、海报初稿、全新产品视觉
编辑底图大体正确，只差局部修正改文字、换对象细节、调色、简化版式
复审后重生成整个概念都错了视角错误、主体错误、层级崩坏

简单说：思路对了但细节错了，就编辑；思路都错了，再重生成。

四、三类核心场景

1. 海报与广告版式

  • 先把主标题定稿，图中只放一行主标题 + 可选副标题
  • 写清文字位置和留白区域
  • 明确投放渠道（社交广告、落地页 Hero 等）

2. 信息图与解释图

  • 结构比"炫"更重要
  • 明确：几个标注区、是否要箭头/分区/编号
  • 像信息设计师一样思考，而非概念画师

3. 产品视觉与品牌画面

  • 镜头角度、表面材质、主光方向
  • 反射状态、背景复杂度、最终投放渠道
  • 这些约束把"普通产品图"推向"可用主视觉图"

海报 Prompt 示例

Create a clean campaign poster for an AI design tool. Centered editorial layout with one product object and generous whitespace. Headline at the top: "Design Faster with Better Control". Small subheading below: "Prompt, edit, and export in one workflow". Soft gray background, silver-blue accents, premium modern typography. Make it suitable for a social ad and a landing-page hero crop.

五、审图清单

第一张图不是终点，而是最快的诊断节点。至少检查：

  • 计划发布尺寸下，文字真的清楚吗？
  • 视觉焦点是不是先落到正确主体？
  • 背景是在帮你表达，还是在抢注意力？
  • 光线与材质是否符合品牌气质？
  • 裁成目标渠道尺寸后还成立吗？
  • 有没有一眼看上去就很假的细节？

六、修改方法

很多糟糕的修改循环，都来自一句空话：make it better。

更好的指令应该像这样：

  • 放大主标题，减少背景细节
  • 保持构图不变，把瓶盖换成拉丝铝
  • 保留信息图结构，压缩调色盘
  • 保持人像光线，去掉多余饰品

一轮只修一类问题，控制力才会回来。

七、常见错误

  1. 把关键词堆砌当成 Prompt
  2. 在图里塞太多文字
  3. 把海报、产品图、信息图、编辑视觉混成一个目标
  4. 明明只需局部修正，却不断从零重生成

核心原则：追求"更可控"，而非"更新奇"。

通用提示词编写指南

一、提示词基本结构

一条好的图像生成提示词通常包含以下五个层次，按重要性从高到低排列：

主体 + 场景 + 风格 + 细节 + 技术参数
  • 主体：画面核心内容 — 谁、什么、几个、什么姿态
  • 场景：环境背景 — 室内/室外、天气、时间、地点
  • 风格：艺术形式 — 摄影写实、3D渲染、插画、油画、海报等
  • 细节：材质、色彩、光影、情绪氛围、服装道具
  • 技术参数：画面比例、画质参数、相机参数等

二、主体描述要点

  • 具体明确：与其写"一个人"，不如写"一位30岁的亚洲女性，短发，戴黑框眼镜，穿白色实验服"
  • 数量清晰：明确主体数量 — "三只猫"而非"几只猫"
  • 动作姿态：描述主体的动作 — "站立、双手交叉、微笑看向镜头"
  • 面部与表情：指定表情 — "温柔微笑"、"专注凝视"、"惊讶张嘴"
  • 服装造型：服装风格决定画面基调 — "街头潮流"与"晚礼服"截然不同

三、场景与构图

景别选择

  • 特写（close-up）：突出面部或物体细节
  • 中景（medium shot）：腰部以上，适合人像
  • 全景（full shot）：全身，展示完整人物
  • 远景（wide shot）：大场景，人物占比小

拍摄角度

  • 平视（eye level）：自然视角
  • 俯拍（bird's-eye / high angle）：从上往下
  • 仰拍（low angle）：从下往上，显大或威严
  • 倾斜（Dutch angle）：制造不安或动感

四、风格与媒介

选择明确的视觉风格，让AI更好地理解你的意图：

摄影类

  • photorealistic / realistic photo
  • cinematic / film still
  • portrait photography
  • product photography
  • documentary style
  • vintage / film grain

艺术类

  • illustration / digital art
  • oil painting / watercolor
  • anime / manga style
  • 3D render / CGI
  • pixel art / vector art
  • line art / sketch

五、光影与色彩

  • 光源类型：sunlight（日光）、studio light（棚拍灯）、neon light（霓虹灯）、candlelight（烛光）、rim light（轮廓光）
  • 光源方向：front lighting（正面光）、side lighting（侧光）、backlighting（逆光）、top lighting（顶光）
  • 光影效果：soft shadows（柔阴影）、hard shadows（硬阴影）、chiaroscuro（明暗对比）、silhouette（剪影）
  • 色调方案：warm tones（暖色调）、cool tones（冷色调）、pastel colors（粉彩色调）、monochrome（单色）、vibrant（鲜艳）、muted（低饱和）
  • 特殊效果：bokeh（背景虚化光斑）、lens flare（镜头光晕）、motion blur（运动模糊）、shallow depth of field（浅景深）

六、常用技术参数

参数说明示例
--ar画面宽高比--ar 16:9（横屏）、--ar 3:4（竖屏）、--ar 1:1（方形）
--v模型版本--v 6.0
--style风格强度--style raw（减少美化）
--q画质--q 2（高清）
--no排除元素--no text, watermark

七、编写技巧

  1. 用正面描述代替负面描述：与其写"不要模糊"，不如写"sharp focus, highly detailed"
  2. 关键词顺序有影响：越靠前的词权重越高
  3. 避免矛盾描述：不要同时要求"realistic photo"和"cartoon style"
  4. 一次不要贪多：一条 prompt 聚焦一个核心场景
  5. 反复迭代：根据结果调整关键词和权重
  6. 保留英文关键词：风格、技术术语、构图参数等保留英文原词
  7. 善用括号加权：部分模型支持 (important detail:1.2) 语法提高特定元素权重

八、示例对比

❌ 不推荐

一只猫 / a cat

过于简单，AI 无法理解具体需求

✅ 推荐

一只橘色虎斑猫坐在洒满阳光的窗台上，窗外是秋天的落叶，温暖的午后光线从左侧照入，柔和的浅景深虚化背景，写实摄影风格，竖构图 3:4，bokeh 效果
